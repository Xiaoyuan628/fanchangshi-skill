## 它是什么？

**反常识**是一个惯性思维破局技能。当你面对一个"大家都这样想"的主流观点、默认方案或直觉认知时，它帮你从第三方视角重新审视——不否定也不肯定，而是提供被忽略的替代视角。

它不是杠精工具。它的目的是**制造一个停顿点**，强制拉出心智带宽，重新审视那些被大脑节能机制跳过的部分。

## 核心框架：四步破局

### 第一步：锚定现状
> "现在的主流/默认/直觉是什么？"

认真还原主流观点的最强形态，不设稻草人。把对手放到最强位置，再开始分析。

### 第二步：拆解底盘
> "它为什么成了默认？靠什么维持着？"

逐层拆解：历史路径、维持机制、隐含前提、有效边界。搞清楚"理所当然"背后靠什么活着。

### 第三步：反直觉视角
> "如果换个角度，会看到什么？"

从经济学、心理学、历史、技术、反面、边缘等维度切入，根据话题类型选2-3个最相关的深入，提供替代性思考。

### 第四步：灰度判断
> "不是对或错，而是在什么条件下更靠谱。"

每个判断都有论据支撑——事实数据、历史先例、逻辑推演。标注可信度光谱，给出关键变量。

| 场景 | 示例 |

## 设计亮点

### 前置判断
不是所有问题都需要反直觉分析。技能会先判断问题类型，走不同的路径：科学事实、情绪化提问、模糊问题、标准争议——每种场景有对应的处理方式。

### 用户选择深度
收到问题后，询问用户选择**快速模式**（300-500字）或**详细模式**（1200-2000字）。不替用户做决定。

### 维度选择策略
不要6个维度全展开。根据话题类型自动选择最相关的2-3个维度深入，避免输出冗长。

### 灰度论证
每个判断不是简单贴标签，而是包含：支撑论据、反面论据、关键变量。让用户自己做出判断。

## 触发词

`反直觉` `质疑` `默认` `理所当然` `大家都这样` `惯性思维` `还有别的可能吗` `为什么是这样` `换个角度想` `主流观点对吗`

## 底线原则

1. **尊重主流**：主流之所以是主流，往往有它的道理
2. **不为反叛而反叛**：目的是找到更优解，不是标新立异
3. **标注边界**：每个观点都说明适用条件
4. **承认不确定性**：证据不足时直接说"不确定"
5. **保持谦逊**：这个技能本身也是一种视角，不是真理

## 灵感来源

这个技能不是在说"从来如此就是错的"，而是在说——从来如此，值得再看一眼。
